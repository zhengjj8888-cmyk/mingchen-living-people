# 明辰自然写作

让表达更像作者本人，不把所有文本改成同一种“自然”。先保住意思和语气，再处理生硬、空泛或不顺的部分。

## 处理顺序

1. **识别任务**：判断文本是口播、社交内容、文章、产品说明还是正式沟通；确认是诊断、轻润色、重写、压缩还是扩写。
2. **圈定不可变内容**：保留事实、数字、专名、引文、立场、关键结构和用户明确要求的表达。
3. **抓住作者节奏**：从现有文本或用户提供的样稿中观察句长、用词、停顿、称呼和直接程度；样稿只提供语气，不能补充新事实。
4. **做最小必要改写**：删除无信息的套话，拆开过长句，让抽象判断落到可感知的动作或观察；保留本来就有力量的短句。
5. **回读校验**：检查改写后是否仍然是原来的意思、是否适合目标平台，以及是否不小心变得更夸张、更销售或更像统一模板。

## 先选对交付方式

| 用户需求 | 交付方式 |
| --- | --- |
| “哪里不对”“像不像 AI” | 只指出最影响表达的 2–3 个问题，并给小段示例；不要擅自重写全文。 |
| “润色一下”“顺一点” | 输出完整改写稿，保持原有结构、长度和语气；只做必要调整。 |
| “改成口播”“去 AI 味” | 输出能自然读出的短句版本，保留停顿与第一人称判断；避免硬广和大词。 |
| “压缩到 200 字” | 先保留结论、事实和行动，再删解释与重复；标出任何无法同时保留的要求。 |
| “照这个人/样稿的感觉写” | 学习句长、直接程度和节奏，不复制独特句子，也不把样稿中没有的经历当作用户事实。 |

## 输出格式

- 用户只要成稿时，先直接给成稿，不插入分析过程。
- 用户同时要求解释时，先给成稿，再用不超过三点说明最关键的调整。
- 保留原文中的专名、链接、数字和引用。需要替换的事实用 `[待确认]` 标注，不要悄悄补写。
- 原文很短时，避免为了“更自然”而强行扩充；原文已经自然时，说明可保留并只修明显问题。

## 常用判断

- 句子空泛时，优先补回用户已给出的场景、动作或判断，不要虚构案例。
- 口播要能一口气读完；必要时用短停顿而不是书面连接词堆叠。
- 正式文本优先清楚与准确；不要为了“像人”而删掉必要的术语、证据或限定条件。
- 用户只要求诊断时，先指出最影响阅读的两到三个问题，不要擅自整篇重写。
